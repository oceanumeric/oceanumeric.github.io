Example 1: verify a claim

Example 2: approximate counting

We are using this example to explain the difference between deterministic models and stochastic models, which is relevant for the example 4 in the coming section. This section is also connected with deterministic and stochastic optimization problems.

Approximate Counting algorithms are techniques that allow us to count a large number of events using a very small amount of memory. It was invented by Robert Morris in 1977 and was published through his paper Counting large number of events in small registers. The algorithm uses probabilistic techniques to increment the counter, although it does not guarantee the exactness it does provide a fairly good estimate of the true value while inducing a minimal and yet fairly constant relative error.

Here is the problem:

  • we need to counter a large number, say 1,000,0001,000,0001,000,000
  • however, our register is only 8-bits, which means 28=2562^8 = 25628=256
  • how could count this large number of events and store the value?

The idea is very simple: instead of counting one by one, we could count 1 after 10 times of even or log⁡2(1+n)\log_2(1+n)log2​(1+n). This means we will have a function to map nnn to XnX_nXn​ like the following one:

Xn=log⁡2(1+n) X_n = \log_2(1 + n) Xn​=log2​(1+n)

The tradeoff is to lose accuracy to count a large number :

  • log⁡2(1+0)=0\log_2(1+0) = 0log2​(1+0)=0, we start from 0
  • log⁡2(1+100)=6.64\log_2(1+100) = 6.64log2​(1+100)=6.64, we may count it as 6 or 7
  • log⁡2(1+120)=6.91\log_2(1+120) = 6.91log2​(1+120)=6.91, we may still count it as 6 or 7
  • log⁡2(1+130)=7.02\log_2(1+130) = 7.02log2​(1+130)=7.02, we may now count it as 7

Therefore the accuracy rate of the above algorithm depends on how we round our errors. We could design a rule as follows:

  • round up if value of the first decimal digit is greater or equal than 555
  • round down otherwise

This kind of algorithm is called deterministic as the rule is determined pre-running and the error was also introduced deterministically. However, we could also introduce the error stochastically as a random variable following some distribution.

Now, again the value XnX_nXn​ stored in a counter is

Example 3: unit testing

Unit tests are typically automated tests written and run by software developers to ensure that a section of an application (known as the “unit”) meets its design and behaves as intended. In procedural programming, a unit could be an entire module, but it is more commonly an individual function or procedure.

Let’s say that you have a dataset with millions of rows and hundreds of variables as columns. Now, you need to write a function to go through each row and do some transformation column wise. For example, suppose you need to write a function to separate year and month from publn_date in Table 1. As you can see that the format is different. Since the dataset is big, there is no way to go through every cell to check the format and write a function to do the separate. What should we do? We rely on probability reasoning and sampling again.

Let’s transform this problem as a probabilistic one:

  • population size (number of rows of dataset): N
  • size of anomalies: A
    • this means you function could cover N−AN-AN−A cases
    • but for those anomalies with size of AAA, the function will not pass the unit testing
  • random sample size: m
    • every time you run the unit testing, you draw a sample of size mmm randomly and run the unit testing
  • Question:
    • scenario i: we will only test once with sample size of mmm, we want to find out the optimal size of mmm with parameters AAA and NNN
    • scenario ii: let’s fix m=5m=5m=5, we want to know how confident we are if we see our code passes the unit testing after 333 trials

Scenario 1

For scenario i, we could model it as a hypergeometric distribution problem. Before we solve our problem, let’s review a classical example first. If we have an urn filled with www white and bbb black balls, then drawing mmm balls out of the urn with replacement, then it yields a Binomial distribution Bin(m,w/(w+b))\mathrm{Bin}(m, w/(w+b))Bin(m,w/(w+b)) for the number of white balls obtained in mmm trials, since the draws are independent Bernoulli trials, each with probability w/(w+b)w/(w+b)w/(w+b) of success (Blitzstein & Hwang, 2015).

Remark: when I tried to model the unit testing problem, I was confused whether I should treat my unit testing as Bernoulli trial or not. The way of thinking about it is to think about:

  • what is an event?
  • how do you define a trial?
  • does your problem fits with Bernoulli distribution? (meaning the success rate and failure rate should be fixed for each trial)

For our unit testing, it is true that each unit test is a trial with success rate and failure rate. However, those rates are not known to us and they are not fixed either. Therefore, we cannot model it as Bernoulli trial.

Now, back to our white and black ball problem, if we instead sample without replacement, then it will not follow Bernoulli distribution. Therefore, we need to do probabilistic reasoning. The population size is N=w+bN = w + bN=w+b.   • how many ways of drawing kkk white balls out of NNN then? It is

(wk)(N−kb) \binom{w}{k} \binom{N-k}{b} (kw​)(bN−k​)

Think about this way: you have 3 ways of cooking food A and 4 ways of cooking food B, how many ways of cooking food ABABAB then?

Example 4: hash tables

This section assumes readers know what hash table is. If not, just check out this video from MIT.

The hash function h:U→[n]h: U \to [n]h:U→[n] maps the elements of universe or population to indices 1,⋯ ,n1, \cdots, n1,⋯,n of an array. The data structure dict from Python is built on hash tables as this kind of map makes it easy to extract information.

Figure 8. Illustration of hash table for dict structure of Python.

Why could not use array to build the dictionary, one array for keys and one array for values and both arrays are mapped with the same index. If we implement this idea, then we need to figure out a way of extracting the value based on the key, say dict_foo["B"]. If all keys are numbers, then dictionary is equivalent to array. However, very often keys are strings. How could achieve O(1)O(1)O(1) performance for finding dict_foo["B"]? A hash table could do that by mapping keys into the indexes of an array (for further explanation).

A hashing function could be either deterministic one (Python implemented it) or stochastic one. We will focus on the stochastic one.

For a big dataset of ∣U∣>>n|U| >> n∣U∣>>n, we need to reply on probability to map elements from universe to the array as the it is economically efficient. For instance, big firms like Google or Facebook are always trying to use less servers or database to serve their customs. In this case the population size is much larger than the number of servers or database.

Figure 9. Illustration of hash table for IP addresses, figure was taken from Musco's course (2022).

As we have explained that it costs a lot to have an index for each element shown in Figure 9. To use less indexes in our hash table, we need to find a way to avoid collisions.

Remark: the application of hashing in data science and data structure like dict is slightly different. For building a data structure like dict, the focus is on achieving O(1)O(1)O(1) performance of finding the value based on the key, whereas for managing a large number of visiting to server the focus is more on mapping between mmm and nnn dynamically.

Now, we store mmm items from a large universe in a hash table with nnn positions. When we insert mmm items into the hash table we may have to store multiple items in the same location (typically as a linked list).

Let’s frame our problem as a probabilistic one:

  • mmm: total number of stored items (sample size)
  • nnn: hash table size (population size)
  • xi,xjx_i, x_jxi​,xj​: any pair of store items chosen randomly
  • CCC: total pairwise collisions
  • hhh: random hash table function

Remark: In example 1, the population size n=1,000,000n=1,000,000n=1,000,000 is fixed, whereas the sample size mmm is updated dynamically every time we draw a sample; for example 2, the hash table size nnn is fixed too, whereas total number of stored items is updated dynamically. It is important to know when and how to set up population and sample space. It might be counterintuitive when you see we set the hash table as the population.

General principle for setting up population and sample: Anything is fixed should be treated as the population, whereas anything is updated dynamically should be treated as the sample space that could be mapped with the random variable function.

With the right setup, we can have the following expectation of pairwise collision

E[C]=∑i,j∈[m],i<jE[Ci,j]=(m2)1n=m(m−1)2n(7) \mathbb{E} [C] = \sum_{i, j \in [m], i < j} \mathbb{E} [C_{i, j}]= \binom{m}{2} \frac{1}{n} = \frac{m(m-1)}{2n} \tag{7} E[C]=i,j∈[m],i<j∑​E[Ci,j​]=(2m​)n1​=2nm(m−1)​(7)

For n=4m2n = 4m^2n=4m2 (size of hashing table is much larger than the sample size), we have

E[C]=m(m−1)8m2<m28m2=18(8) \mathbb{E} [C] = \frac{m(m-1)}{8m^2} < \frac{m^2}{8m^2} = \frac{1}{8} \tag{8} E[C]=8m2m(m−1)​<8m2m2​=81​(8)

This means we could have the collision-free hash table if the hash table is large enough. This completely makes sense. However, we do not gain any economic benefit as we are using O(m2)O(m^2)O(m2) space to store mmm elements.

Let’s apply Markov’s inequality in this case, we can have

P(C≥1)≤E(C)1=18P(C=0)=1−P(C≥1)=78(9) \begin{aligned} \mathbb{P} (C \geq 1) \leq \frac{\mathbb{E} (C)}{1} = \frac{1}{8} \\ \mathbb{P}(C = 0) = 1 - \mathbb{P} (C \geq 1) = \frac{7}{8} \tag{9} \end{aligned} P(C≥1)≤1E(C)​=81​P(C=0)=1−P(C≥1)=87​​(9)

Two Level Hashing. Now, if we want to preserve O(1)O(1)O(1) query with O(m)O(m)O(m) space, we can do two level hashing as shown in Figure 5.

Figure 10. Illustration of two level hashing; figure was taken from Musco's course (2022).

In equation 9, we have shown that the random hashing function could guarantee that the collision is avoided at 7/87/87/8 probability level. Therefore, we could just use a random hashing function at level two. Now, let’s set up the scene to do probabilistic thinking:

  • mmm: total number of stored items (sample size)
  • nnn: hash table size (population size)
  • xj,xkx_j, x_kxj​,xk​: any pair of store items chosen randomly
  • CCC: total pairwise collisions
  • hhh: random hash table function
  • SSS: spaced use for the first and second level of hashing
  • sis_isi​: items (plural) stored in the hash table at position iii (see Figure 5)
    • for each bucket in hashing table of size nnn, pick a collision free hash function mapping [si]→[si2][s_i] \to [s_i^2][si​]→[si2​] (as it is shown in equation 8).

What is the expected space usage for two level hashing? It is quite straightforward to calculating the total space usage.

E[S]=n+∑i=1nE[si2](10) \mathbb{E} [S] = n + \sum_{i=1}^n \mathbb{E} [s_i^2] \tag{10} E[S]=n+i=1∑n​E[si2​](10)

Before we calculating E[si2]\mathbb{E} [s_i^2]E[si2​], let’s go through two scenario (as always, we assumed m>>nm > > nm>>n):

  • determined path: for mmm items, we assign nnn of them into our hashing table and then we assign (m−n)(m-n)(m−n) elements into the second level linked list
  • stochastic path: every item was assign to a cell randomly and we will use an indicator function to track them.
    • we don’t know which element will be assigned into the container sis_isi​ (iii is the index of position)
    • but we could use an indicator function to track them

I{si}[h(xj)]:={1h(xj)=i0h(xj)≠i \mathbb{I}_{\{s_i\}}[h(x_j)] := \begin{cases} 1 & h(x_j) = i \\ 0 & h(x_j) \neq i \end{cases} I{si​}​[h(xj​)]:={10​h(xj​)=ih(xj​)​=i​

Therefore, we could have

E[si2]=E[(∑j=1mIh(xj)=i)2](11) \mathbb{E}[s_i^2] = \mathbb{E} \left [ \left ( \sum_{j=1}^m \mathbb{I}_{h(x_j)=i} \right )^2 \right] \tag{11} E[si2​]=E⎣⎢⎡​(j=1∑m​Ih(xj​)=i​)2⎦⎥⎤​(11)

Equation (11) is saying that we sum up all possible elements assigned into sis_isi​ for element xj,j∈[m]x_j, j \in [m]xj​,j∈[m]. Now, let’s expand the equation 11,

E[si2]=E[(∑j=1mIh(xj)=i)2]=E[(∑j=1mIh(xj)=i)(∑j=1mIh(xj)=i)]=E[∑j,k∈[m]Ih(xj)=i⋅Ih(xk)=i]=∑j,k∈[m]E [Ih(xj)=i⋅Ih(xk)=i](12) \begin{aligned} \mathbb{E}[s_i^2] & = \mathbb{E} \left [ \left ( \sum_{j=1}^m \mathbb{I}_{h(x_j)=i} \right )^2 \right] \\ & = \mathbb{E} \left [ \left ( \sum_{j=1}^m \mathbb{I}_{h(x_j)=i} \right ) \left ( \sum_{j=1}^m \mathbb{I}_{h(x_j)=i} \right ) \right] \\ & = \mathbb{E} \left[ \sum_{j, k \in [m]} \mathbb{I}_{h(x_j)=i} \cdot \mathbb{I}_{h(x_k)=i} \right ] \\ & = \sum_{j, k \in [m]} \mathbb{E} \ \Bigl[\mathbb{I}_{h(x_j)=i} \cdot \mathbb{I}_{h(x_k)=i} \Bigr] \tag{12} \end{aligned} E[si2​]​=E⎣⎢⎡​(j=1∑m​Ih(xj​)=i​)2⎦⎥⎤​=E[(j=1∑m​Ih(xj​)=i​)(j=1∑m​Ih(xj​)=i​)]=E⎣⎢⎡​j,k∈[m]∑​Ih(xj​)=i​⋅Ih(xk​)=i​⎦⎥⎤​=j,k∈[m]∑​E [Ih(xj​)=i​⋅Ih(xk​)=i​]​(12)

The second step of equation (12) is derived because many values of indicator function are zeros and the last step is derived due to the linearity of expectation.

With equation (12), we need to calculate the expectation

∑j,k∈[m]E [Ih(xj)=i⋅Ih(xk)=i]\sum_{j, k \in [m]} \mathbb{E} \ \Bigl [\mathbb{I}_{h(x_j)=i} \cdot \mathbb{I}_{h(x_k)=i} \Bigr ] j,k∈[m]∑​E [Ih(xj​)=i​⋅Ih(xk​)=i​]

We will do this by calculating two cases:

  • j=k j = kj=k
  • j≠kj \neq kj​=k

For j=kj=kj=k, the derivation is simple

E [Ih(xj)=i⋅Ih(xk)=i]=E[(Ih(xj)=i)2]=P[(h(xj)=i)]=1n \mathbb{E} \ \Biggl [\mathbb{I}_{h(x_j)=i} \cdot \mathbb{I}_{h(x_k)=i} \Biggr] = \mathbb{E} \left[ \left (\mathbb{I}_{h(x_j)=i} \right)^2 \right] = \mathbb{P}[(h(x_j) = i)] = \frac{1}{n} E [Ih(xj​)=i​⋅Ih(xk​)=i​]=E[(Ih(xj​)=i​)2]=P[(h(xj​)=i)]=n1​

For j≠kj \neq kj​=k, we have

E [Ih(xj)=i⋅Ih(xk)=i]=P[(h(xj)=i)∩h(xk)=i]=1n2 \mathbb{E} \ \Biggl [\mathbb{I}_{h(x_j)=i} \cdot \mathbb{I}_{h(x_k)=i} \Biggr] = \mathbb{P}[(h(x_j) = i) \cap h(x_k) = i] = \frac{1}{n^2} E [Ih(xj​)=i​⋅Ih(xk​)=i​]=P[(h(xj​)=i)∩h(xk​)=i]=n21​

Therefore, according to the linearity of expectation, we have

E[si2]=∑j,k∈[m]E [Ih(xj)=i⋅Ih(xk)=i]=m⋅1n+2⋅(m2)1n2=mn+m(m−1)n2(13) \begin{aligned} \mathbb{E}[s_i^2] & = \sum_{j, k \in [m]} \mathbb{E} \ \Bigl[\mathbb{I}_{h(x_j)=i} \cdot \mathbb{I}_{h(x_k)=i} \Bigr] \\ & = m \cdot \frac{1}{n} + 2 \cdot \binom{m}{2} \frac{1}{n^2} \\ & = \frac{m}{n} + \frac{m(m-1)}{n^2} \tag{13} \end{aligned} E[si2​]​=j,k∈[m]∑​E [Ih(xj​)=i​⋅Ih(xk​)=i​]=m⋅n1​+2⋅(2m​)n21​=nm​+n2m(m−1)​​(13)

When we set n=mn = mn=m in equation (13), we can get

E[si2]=mn+m(m−1)n2<2(14) \mathbb{E}[s_i^2] = \frac{m}{n} + \frac{m(m-1)}{n^2} < 2 \tag{14} E[si2​]=nm​+n2m(m−1)​<2(14)

Now, with this bound, we can substitute the equation (14) into equation (10) and have our final result,

E[S]=n+∑i=1nE[si2]≤n+∑i=1n2=3n=3m(15) \mathbb{E} [S] = n + \sum_{i=1}^n \mathbb{E} [s_i^2] \leq n + \sum_{i=1}^n 2 = 3n = 3m \tag{15} E[S]=n+i=1∑n​E[si2​]≤n+i=1∑n​2=3n=3m(15)

This means we achieved a near optimal space with O(1)O(1)O(1) query time!
