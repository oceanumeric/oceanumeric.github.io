When we use a random hash function h(x)h(x)h(x) to map keys into hashing numbers, we could design a fully random one which has the following property:

P(h(x)=i)=1n \mathbb{P}(h(x) = i) = \frac{1}{n} P(h(x)=i)=n1​

However, wit the above property, we need to store a table of xxx values and their hash values. This would take at least O(m)O(m)O(m) space and O(m)O(m)O(m) query time to look up, making our whole quest for O(1)O(1)O(1) query time pointless. Therefore we need to find another property that will avoid collision and O(1)O(1)O(1) query time.

2-Universial Hash Function (low collision probability). A random hash function from h:U→[n]h: U \to [n]h:U→[n] is two universal if :

P[h(x)=h(y)]≤1n(1) \mathbb{P}[h(x) = h(y)] \leq \frac{1}{n} \tag{1} P[h(x)=h(y)]≤n1​(1)

One of the functions that are often used is the following one:

h(x)=(ax+b modp) mod n h(x) = (a x + b \ \mathrm{mod} p) \ \mathrm{mod} \ n h(x)=(ax+b modp) mod n

Pairwise Independent Hash Function. A random hash function from h:U→[n]h: U \to [n]h:U→[n] is pairwise independence if for all i,j∈[n]i, j \in [n]i,j∈[n]:

P[h(x)=i∩h(y)=j]=1n2(2) \mathbb{P}[h(x) = i \cap h(y) = j] = \frac{1}{n^2} \tag{2} P[h(x)=i∩h(y)=j]=n21​(2)

We can show that pairwise hash function are 2-universal. Now, for i∈[n]i \in [n]i∈[n], it has n−1n-1n−1 pairs and one pair with itself, therefore:

P[h(x)=h(y)]=∑j=1nP[h(x)=i∩h(y)=j]=n⋅1n2=1n \begin{aligned} \mathbb{P}[h(x) = h(y)] & = \sum_{j=1}^n \mathbb{P}[h(x) = i \cap h(y) = j] \\ & = n \cdot \frac{1}{n^2} = \frac{1}{n} \end{aligned} P[h(x)=h(y)]​=j=1∑n​P[h(x)=i∩h(y)=j]=n⋅n21​=n1​​

Equation (5) gives us the quick estimation on deviation of our estimations. It could also shows that the sample average will always concentrate on mean with enough samples nnn (law of large numbers).

Maximum server load

We have shown the probability of one server overloaded in equation (8), now we are interested in the probability of maximum server load exceeds its capacity. To get the maximum server load, we need to have a list of servers that overloaded, this means not just one sever but possible many of them are overloaded. The value of maximum load on any server can be written as the random variable:

S=max⁡i∈[1,⋯ ,k]RiS = \max_{i \in [1, \cdots, k]} R_iS=i∈[1,⋯,k]max​Ri​

How could we form a probability for S≥(2n)/kS \geq (2n)/kS≥(2n)/k in terms of RiR_iRi​? Let’s start from null case, which is that no RiR_iRi​ is greater or equal to (2n)/k(2n)/k(2n)/k. For this kind of case, S<(2n)/kS < (2n)/kS<(2n)/k. Therefore, we need to have at least one of RiR_iRi​ is greater or equal to (2n)/k(2n)/k(2n)/k. On the other side, if all of RiR_iRi​ is greater or equal to (2n)/k(2n)/k(2n)/k, then there must be the case that S≥(2n)kS \geq (2n)kS≥(2n)k. However, we do not need all of them. We can transform them as the ‘or’ problem that guarantees the event SSS:

P(S≥2nk)=P([R1≥2nk] or [R2≥2nk] or ⋯ or [Rk≥2nk])=P(∪i=1k[Ri≥2nk]) \begin{aligned} \mathbb{P}(S \geq \frac{2n}{k}) & = \mathbb{P} \left ( \left[ R_1 \geq \frac{2n}{k} \right ] \ \text{or} \ \left[ R_2 \geq \frac{2n}{k} \right ] \ \text{or} \ \cdots \ \text{or} \ \left[ R_k \geq \frac{2n}{k} \right ] \right ) \\ & = \mathbb{P} \left ( \cup_{i=1}^k \left[ R_i \geq \frac{2n}{k} \right ] \right ) \end{aligned} P(S≥k2n​)​=P([R1​≥k2n​] or [R2​≥k2n​] or ⋯ or [Rk​≥k2n​])=P(∪i=1k​[Ri​≥k2n​])​

Union Bound. For any random events A1,A2,⋯ ,AkA_1, A_2, \cdots, A_kA1​,A2​,⋯,Ak​, we have

P(A1∪A2∪⋯∪Ak)≤P(A1)+P(A2)+⋯+P(Ak)(9) \mathbb{P}(A_1 \cup A_2 \cup \cdots \cup A_k) \leq \mathbb{P}(A_1) + \mathbb{P}(A_2) + \cdots + \mathbb{P}(A_k) \tag{9} P(A1​∪A2​∪⋯∪Ak​)≤P(A1​)+P(A2​)+⋯+P(Ak​)(9)
