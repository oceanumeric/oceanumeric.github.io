Approximating Distinct Element in a Stream

This post explains a probabilistic counting algorithm with which one can estimate the number of distinct elements in a large collection of data in a single pass.

probability, algorithm, data-science, machine-learning, bloom-filter, online-learning

Published

08 March 2023

This post is based on notes by Alex Tsun and paper by Flajolet and Martin (1985). It is pretty amazing the the algorithm discovered in the 1980s when the memory was very expensive has a huge number of applications in networking, database, and computer system now - big data era (Flajolet & Martin, 1985).

Motivation problem

For one of the most popular videos on Youtube, let’s say there are N=2N = 2N=2 billion views, with n=900n = 900n=900 million of them being distinct views. How much space is required to accurately track this number? Well, let’s assume a user ID is an 8-byte integer. Then, we need 900,000,000×8900,000,000 \times 8900,000,000×8 bytes total if we use a Set to track the user IDs, which requires 7.27.27.2 gigabytes of memory for ONE video.

Intuitions and examples

When we count distinct values nnn, we notice that it has the following property:

  • n>=0n >= 0n>=0 for sure
  • nnn is monotonic increasing

For instance, we cannot have −2-2−2 distinct values (distinct users). Furthermore, the distinct values always increase or remain constant, such as we have 777 distinct users at the current moment and this value will either increase or remain constant during the streaming process.

Instead of having a set of distinct values and querying the new value is unique or not, we could rely on hashing and probability to count the number of distinct values with O(1)O(1)O(1) space.

Figure 1. Illustration of using hash function to count the number of distinct values.

The idea of approximating distinct elements in a stream is to use the minimum value generated randomly from the hash function as it is illustrated in the Figure 1.

For the second equation, we use the fact that the minimum of numbers is greater than a value if and only if all of them are.
