Variational Inference (1)

Modern Bayesian statistics relies on models for which the posterior is not easy to compute and corresponding algorithms for approximating them. Variational inference is one of the most popular methods for approximating the posterior. In this post, we will introduce the basic idea of variational inference and its application to a simple example.

Big picture

When Shannon introduced the concept of entropy, he was asking the question: how much is our uncertainty reduced by learning an outcome of a random variable? In other words, how much information do we gain by learning an outcome of a random variable? This is the question that information theory tries to answer.

To measure the information gain is to measure uncertainty. To come up an equation that could measure uncertainty, we need to design some criteria first (McElreath, 2020). The criteria that Shannon used is the following:

  1. The measure should be continuous.
  2. The measure should be additive.
  3. The measure of uncertainty should increase as the number of possible outcomes increases.

The function satisfying the above criteria is the following:

H(X)=−∑i=1np(xi)log⁡p(xi).(1) H(X) = -\sum_{i=1}^n p(x_i) \log p(x_i). \tag{1} H(X)=−i=1∑n​p(xi​)logp(xi​).(1)

This function is called the entropy of a random variable XXX. It measures the uncertainty of a random variable. The more uncertain a random variable is, the higher its entropy is.

Let’s have a simple example to illustrate the idea.

The table above shows three columns with different kinds of ‘information’, and we are using integer to represent each word. This means we could calculate the probability for each variable by setting sample size n=10n = 10n=10. By reading those words, one could see that the third column gives more information as it introduces more uncertainty (or variance). This is also reflected by the entropy of each column as it is shown in the figure below.

After having a sense of entropy, we could move on to the next question: how to measure the distance between two probability distributions? This is the question that Kullback and Leibler (KL) divergence tries to answer. The KL divergence is defined as the following:

DKL(p,q)=∑i=1np(xi)log⁡p(xi)q(xi)=∑i=1np(xi)[log⁡p(xi)−log⁡q(xi)].(2) D_{KL}(p, q) = \sum_{i=1}^n p(x_i) \log \frac{p(x_i)}{q(x_i)} = \sum_{i=1}^n p(x_i) \left [ \log p(x_i) - \log q(x_i) \right ]. \tag{2} DKL​(p,q)=i=1∑n​p(xi​)logq(xi​)p(xi​)​=i=1∑n​p(xi​)[logp(xi​)−logq(xi​)].(2)

Equation (2) is the KL divergence between two probability distributions ppp (the target) and qqq (the model). This equations is often called the cross-entropy between ppp and qqq.

With those two concepts, we will use a concrete example to illustrate the idea of variational inference, which is about expectation maximization (EM) algorithm.

Density estimation

Suppose we have a dataset D={x1,x2,…,xn}\mathcal{D} = \{x_1, x_2, \dots, x_n\}D={x1​,x2​,…,xn​}, and we want to estimate the density of the data. Let’s run a simulation to generate some data and see how it looks like.

import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns


def figure1():
    # set seed

    x1 = np.random.normal(mu1, sigma, n)

with this dataset, we know there are two normal distributions. However, if we don’t know the data generating process, we could only estimate the density of the data.

To classify the data, we could introduce a latent variable zzz to represent the class of the data, z=(z(1),⋯ ,z(m))z = (z^{(1)}, \cdots, z^{(m)})z=(z(1),⋯,z(m)), which is multinomial distributed, and z(i)z^{(i)}z(i) represents the class of the iii-th data point. For our example in figure 2, we have two classes, so z(i)∈{0,1}z^{(i)} \in \{0, 1\}z(i)∈{0,1}, which is a Bernoulli distribution.

Now, we could model the data generating process as the following:

p(x,z;Θ);Θis the parameter (mean, etc.) of the model.(3) p(x, z; \Theta); \quad \Theta \text{is the parameter (mean, etc.) of the model}. \tag{3} p(x,z;Θ);Θis the parameter (mean, etc.) of the model.(3)

Since we only observe the data xxx, we could marginalize the latent variable zzz to get the marginal distribution of xxx:

p(x;Θ)=∑zp(x,z;Θ).(4) p(x; \Theta) = \sum_z p(x, z; \Theta). \tag{4} p(x;Θ)=z∑​p(x,z;Θ).(4)

We could use the maximum likelihood estimation (MLE) to estimate the parameter Θ\ThetaΘ by maximizing the log-likelihood of the data, such as

Θ∗=arg⁡max⁡Θ∏i=1np(x(i);Θ)=arg⁡max⁡Θ∑i=1nln⁡p(x;Θ)=arg⁡max⁡Θ∑i=1nln⁡∑zp(x,z;Θ).(5) \begin{aligned} \Theta^* & = \arg \max_\Theta \prod_{i=1}^n p(x^{(i)}; \Theta) \\ & = \arg \max_\Theta \sum_{i=1}^n \ln p(x; \Theta) \\ & = \arg \max_\Theta \sum_{i=1}^n \ln \sum_z p(x, z; \Theta). \end{aligned} \tag{5} Θ∗​=argΘmax​i=1∏n​p(x(i);Θ)=argΘmax​i=1∑n​lnp(x;Θ)=argΘmax​i=1∑n​lnz∑​p(x,z;Θ).​(5)

To solve this problem directly, we need to calculate the derivative of the log-likelihood with respect to the parameter Θ\ThetaΘ, which is not easy to do (we have to calculate the derivative of the log of the sum of the probability because of the latent variable zzz).

Instead, we could use the EM algorithm to solve this problem. To understand the EM algorithm, we need to introduce the concept of lower bound, which is based on Jensen’s inequality.

Jensen’s inequality

Jensen’s inequality states that for a convex function fff, we have

f(E[x])≤E[f(x)].(6) f(\mathbb{E}[x]) \leq \mathbb{E}[f(x)]. \tag{6} f(E[x])≤E[f(x)].(6)

If the function fff is concave, then we have

f(E[x])≥E[f(x)].(7) f(\mathbb{E}[x]) \geq \mathbb{E}[f(x)]. \tag{7} f(E[x])≥E[f(x)].(7)

Now, let f(x)=ln⁡xf(x) = \ln xf(x)=lnx, which is a concave function, then we have

ln⁡E[x]≥E[ln⁡x].(8) \ln \mathbb{E}[x] \geq \mathbb{E}[\ln x]. \tag{8} lnE[x]≥E[lnx].(8)

EM algorithm

Now, we could use the Jensen’s inequality to derive the EM algorithm. First, we could rewrite the log-likelihood in equation (4) as the following:

ln⁡p(x;Θ)=ln⁡∑zp(x,z;Θ)=ln⁡∑zq(z)p(x,z;Θ)q(z)≥∑zq(z)ln⁡p(x,z;Θ)q(z)(9) \begin{aligned} \ln p(x; \Theta) & = \ln \sum_z p(x, z; \Theta) \\ & = \ln \sum_z q(z) \frac{p(x, z; \Theta)}{q(z)} \\ & \geq \sum_z q(z) \ln \frac{p(x, z; \Theta)}{q(z)} \end{aligned} \tag{9} lnp(x;Θ)​=lnz∑​p(x,z;Θ)=lnz∑​q(z)q(z)p(x,z;Θ)​≥z∑​q(z)lnq(z)p(x,z;Θ)​​(9)

where q(z)q(z)q(z) is a distribution over the latent variable zzz. The last step is based on the Jensen’s inequality. Now, we could substitute the lower bound of the log-likelihood into the MLE problem in equation (5):

L(Θ,q)=∑i=1nln⁡∑zp(x,z;Θ)≥∑i=1n∑zq(z)ln⁡p(x,z;Θ)q(z)(10) \begin{aligned} L(\Theta, q) & = \sum_{i=1}^n \ln \sum_z p(x, z; \Theta) \\ & \geq \sum_{i=1}^n \sum_z q(z) \ln \frac{p(x, z; \Theta)}{q(z)} \end{aligned} \tag{10} L(Θ,q)​=i=1∑n​lnz∑​p(x,z;Θ)≥i=1∑n​z∑​q(z)lnq(z)p(x,z;Θ)​​(10)

This is the lower bound of the log-likelihood. Now, we could maximize the lower bound with respect to the parameter Θ\ThetaΘ and the distribution q(z)q(z)q(z), which is equivalent to maximizing the log-likelihood.

Now, if we fix the supremum of the lower bound, then we could have

p(x,z;Θ)q(z)=const=c(11) \frac{p(x, z; \Theta)}{q(z)} = \text{const} = c \tag{11} q(z)p(x,z;Θ)​=const=c(11)

This leads to

q(z)∝p(x,z;Θ); s.t.∑zq(z)=1.(12) q(z) \propto p(x, z; \Theta); \quad \ s.t. \sum_z q(z) = 1. \tag{12} q(z)∝p(x,z;Θ); s.t.z∑​q(z)=1.(12)

This shows that the distribution q(z)q(z)q(z) is the posterior distribution of the latent variable zzz given the data xxx and the parameter Θ\ThetaΘ. Therefore, we could have

p(z∣x;Θ)=p(x,z;Θ)p(x;Θ)=p(x,z;Θ)∑zp(x,z;Θ)=q(z).(13) p(z|x; \Theta) = \frac{p(x, z; \Theta)}{p(x; \Theta)} = \frac{p(x, z; \Theta)}{\sum_z p(x, z; \Theta)} = q(z). \tag{13} p(z∣x;Θ)=p(x;Θ)p(x,z;Θ)​=∑z​p(x,z;Θ)p(x,z;Θ)​=q(z).(13)

This is the E-step of the EM algorithm. In the E-step, we calculate the posterior distribution of the latent variable zzz given the data xxx and the parameter Θ\ThetaΘ.

For the M-step, we maximize the lower bound with respect to the parameter Θ\ThetaΘ:

Θ∗=arg⁡max⁡Θ∑i=1nq(z)ln⁡∑zp(x,z;Θ)q(z)(14) \Theta^* = \arg \max_\Theta \sum_{i=1}^n q(z) \ln \frac{\sum_z p(x, z; \Theta)}{q(z)} \tag{14} Θ∗=argΘmax​i=1∑n​q(z)lnq(z)∑z​p(x,z;Θ)​(14)

Before we implement the EM algorithm, we will link the EM algorithm to KL divergence in equation (2).

Evidence Lower Bound (ELBO)

We could rewrite the lower bound in equation (9) as the following:

ln⁡p(x;Θ)=∑zq(z)ln⁡p(x,z;Θ)q(z)=∑zq(z)ln⁡p(x,z;Θ)p(z∣x;Θ)=∑zq(z)ln⁡p(x,z;Θ)/q(z)p(z∣x;Θ)/q(z)=∑zq(z)ln⁡p(x,z;Θ)q(z)−∑zq(z)ln⁡p(z∣x;Θ)q(z)=∑zq(z)ln⁡p(x,z;Θ)q(z)+∑zq(z)ln⁡q(z)p(z∣x;Θ)=L(x,Θ)+KL(q(z)∣∣p(z∣x;Θ))(15) \begin{aligned} \ln p(x; \Theta) &= \sum_{z} q(z) \ln \frac{p(x, z; \Theta)}{q(z)} \\ &= \sum_{z} q(z) \ln \frac{p(x, z; \Theta)}{p(z|x; \Theta)} \\ &= \sum_{z} q(z) \ln \frac{p(x, z; \Theta)/q(z)}{p(z|x; \Theta)/q(z)} \\ & = \sum_{z} q(z) \ln \frac{p(x, z; \Theta)}{q(z)} - \sum_{z} q(z) \ln \frac{p(z|x; \Theta)}{q(z)} \\ & = \sum_{z} q(z) \ln \frac{p(x, z; \Theta)}{q(z)} + \sum_{z} q(z) \ln \frac{q(z)}{p(z|x; \Theta)} \\ & = L(x, \Theta) + KL(q(z) || p(z|x; \Theta)) \end{aligned} \tag{15} lnp(x;Θ)​=z∑​q(z)lnq(z)p(x,z;Θ)​=z∑​q(z)lnp(z∣x;Θ)p(x,z;Θ)​=z∑​q(z)lnp(z∣x;Θ)/q(z)p(x,z;Θ)/q(z)​=z∑​q(z)lnq(z)p(x,z;Θ)​−z∑​q(z)lnq(z)p(z∣x;Θ)​=z∑​q(z)lnq(z)p(x,z;Θ)​+z∑​q(z)lnp(z∣x;Θ)q(z)​=L(x,Θ)+KL(q(z)∣∣p(z∣x;Θ))​(15)

Where L(x,Θ)L(x, \Theta)L(x,Θ) is the lower bound of the log-likelihood and KL(q(z)∣∣p(z∣x;Θ))KL(q(z) || p(z|x; \Theta))KL(q(z)∣∣p(z∣x;Θ)) is the KL divergence between the posterior distribution q(z)q(z)q(z) and the true posterior distribution p(z∣x;Θ)p(z|x; \Theta)p(z∣x;Θ).

L(x,Θ)L(x, \Theta)L(x,Θ) is also called the evidence lower bound (ELBO). The ELBO is a lower bound of the log-likelihood. The KL divergence is always non-negative, which means that the ELBO is always smaller than the log-likelihood.

ln⁡p(x;Θ)≥L(x,Θ)(16) \ln p(x; \Theta) \geq L(x, \Theta) \tag{16} lnp(x;Θ)≥L(x,Θ)(16)

Therefore, we could maximize the ELBO to maximize the log-likelihood.

Application of EM algorithm

Now, we could apply the EM algorithm to Gaussian mixture model (GMM). Suppose we have some data x1,x2,⋯ ,xnx_1, x_2, \cdots, x_nx1​,x2​,⋯,xn​, which is from KKK Gaussian distributions (K mixture components). To estimate the parameters of the GMM, we could use the EM algorithm.

Let’s set up our notation first:

  • μk\mu_kμk​ is the mean of the kkk-th Gaussian distribution.
  • Σk\Sigma_kΣk​ is the covariance matrix of the kkk-th Gaussian distribution.
  • ϕk\phi_kϕk​ is the mixing coefficient of the kkk-th Gaussian distribution.
  • ziz_izi​ is the latent variable of the iii-th data point. ziz_izi​ is a one-hot vector, which means that zik=1z_{ik} = 1zik​=1 if the iii-th data point is from the kkk-th Gaussian distribution. Otherwise, zik=0z_{ik} = 0zik​=0.

Remark: xix_ixi​ does not have to be a scalar. It could be a vector such as xi∈Rdx_i \in \mathbb{R}^dxi​∈Rd.

Our goal is to maximize the log-likelihood of the GMM:

arg⁡max⁡μ,Σ,ϕ∑i=1nln⁡p(xi;μ,Σ,ϕ)(17) \arg \max_{\mu, \Sigma, \phi} \sum_{i=1}^n \ln p(x_i; \mu, \Sigma, \phi) \tag{17} argμ,Σ,ϕmax​i=1∑n​lnp(xi​;μ,Σ,ϕ)(17)

E-step: In the E-step, we calculate the posterior distribution of the latent variable zzz given the data xxx and the parameter Θ\ThetaΘ:

q(zi)=p(zi∣xi;Θ)=p(zi∣xi;μ,Σ,ϕ)(18) q(z_i) = p(z_i | x_i; \Theta) = p(z_i | x_i; \mu, \Sigma, \phi) \tag{18} q(zi​)=p(zi​∣xi​;Θ)=p(zi​∣xi​;μ,Σ,ϕ)(18)

M-step: In the M-step, we maximize the lower bound with respect to the parameter Θ\ThetaΘ. The reason why we could maximize the lower bound is that the KL divergence is always non-negative. Therefore, we could maximize the lower bound to maximize the log-likelihood, which makes the optimization process tractable.

According to equation (14), we could maximize the lower bound with respect to the parameter Θ\ThetaΘ:

Θ∗=arg⁡max⁡Θ∑i=1n∑jKq(zi=j)ln⁡p(xi,zi;Θ)qi(zi=j)=arg⁡max⁡Θ∑i=1n∑jKq(zi=j)ln⁡p(xi∣zi=j;μ,Σ)p(zi=j;ϕ)qi(zi=j)(19) \begin{aligned} \Theta^* & = \arg \max_\Theta \sum_{i=1}^n \sum_{j}^K q(z_i= j) \ln \frac{p(x_i, z_i ; \Theta)}{q_i(z_i = j)} \\ & = \arg \max_\Theta \sum_{i=1}^n \sum_{j}^K q(z_i= j)\ln \frac{p(x_i|z_i = j; \mu, \Sigma)p(z_i = j; \phi)}{q_i(z_i = j)} \\ \end{aligned} \tag{19} Θ∗​=argΘmax​i=1∑n​j∑K​q(zi​=j)lnqi​(zi​=j)p(xi​,zi​;Θ)​=argΘmax​i=1∑n​j∑K​q(zi​=j)lnqi​(zi​=j)p(xi​∣zi​=j;μ,Σ)p(zi​=j;ϕ)​​(19)

With the above format, we could leverage the distribution functions of xix_ixi​ and ziz_izi​ to calculate the lower bound because

xi∣zi=j;μ,Σ∼N(μj,Σj)zi∼Categorical(ϕ)(20) \begin{aligned} x_i|z_i = j; \mu, \Sigma & \sim \mathcal{N}(\mu_j, \Sigma_j) \\ z_i & \sim \text{Categorical}(\phi) \end{aligned} \tag{20} xi​∣zi​=j;μ,Σzi​​∼N(μj​,Σj​)∼Categorical(ϕ)​(20)

Therefore, the equation (19) could be rewritten as:

L:=∑i=1n∑j=1Kwj(i)ln⁡1(2π)n∣Σj∣exp⁡[−12(xi−μj)′Σj−1(xi−μj)]ϕjwj(i)(21) L := \sum_{i=1}^n \sum_{j=1}^K w_j^{(i)} \ln \frac{\frac{1}{\sqrt{(2\pi)^n|\Sigma_j|}} \exp \left[ -\frac{1}{2}(x_i - \mu_j)'\Sigma_j^{-1}(x_i - \mu_j) \right] \phi_j}{w_j^{(i)}} \tag{21} L:=i=1∑n​j=1∑K​wj(i)​lnwj(i)​(2π)n∣Σj​∣​1​exp[−21​(xi​−μj​)′Σj−1​(xi​−μj​)]ϕj​​(21)

With the above equation, we could take the derivative of llllll with respect to μj\mu_jμj​, Σj\Sigma_jΣj​, and ϕj\phi_jϕj​ and set the derivative to zero to find the optimal parameters.

First, let’s take the derivative of LLL with respect to μj\mu_jμj​:

∂L∂μj=∑i=1n∂∂μj(ln⁡1(2π)n∣Σj∣wji+[−12(xi−μj)′Σj−1(xi−μj)])=∑i=1nwj(i)∂∂μj[−12(xi−μj)′Σj−1(xi−μj)]=12∂∂μj∑i=1nwj(i)[(Σj−1+(Σj−1)′(xi−μj))]=∑i=1nwj(i)Σj−1(xi−μj)=Σj−1∑i=1nwj(i)(xi−μj)=0(22) \begin{aligned} \frac{\partial L}{\partial \mu_j} & = \sum_{i=1}^n \frac{\partial}{\partial \mu_j} \left ( \ln \frac{\frac{1}{\sqrt{(2\pi)^n|\Sigma_j|}}}{w_j^{i}} + \left[ -\frac{1}{2}(x_i - \mu_j)'\Sigma_j^{-1}(x_i - \mu_j) \right] \right ) \\ & = \sum_{i=1}^n w_j^{(i)} \frac{\partial}{\partial \mu_j} \left[ -\frac{1}{2}(x_i - \mu_j)'\Sigma_j^{-1}(x_i - \mu_j) \right] \\ & = \frac{1}{2} \frac{\partial}{\partial \mu_j} \sum_{i=1}^n w_j^{(i)} \left [ (\Sigma_j^{-1} + (\Sigma_j^{-1})'(x_i - \mu_j)) \right] \\ & = \sum_{i=1}^n w_j^{(i)} \Sigma_j^{-1}(x_i - \mu_j) \\ & = \Sigma_j^{-1} \sum_{i=1}^n w_j^{(i)} (x_i - \mu_j) \\ & = 0 \end{aligned} \tag{22} ∂μj​∂L​​=i=1∑n​∂μj​∂​⎝⎜⎛​lnwji​(2π)n∣Σj​∣​1​​+[−21​(xi​−μj​)′Σj−1​(xi​−μj​)]⎠⎟⎞​=i=1∑n​wj(i)​∂μj​∂​[−21​(xi​−μj​)′Σj−1​(xi​−μj​)]=21​∂μj​∂​i=1∑n​wj(i)​[(Σj−1​+(Σj−1​)′(xi​−μj​))]=i=1∑n​wj(i)​Σj−1​(xi​−μj​)=Σj−1​i=1∑n​wj(i)​(xi​−μj​)=0​(22)

This gives us the following equation:

Implementation in Python

Now, let’s implement the EM algorithm in Python. If you look at the equations above, you should notice that we have to calculate wj(i)w_j^{(i)}wj(i)​ in each iteration, which is the probability that xix_ixi​ belongs to the jjj-th Gaussian distribution. This is the responsibility of the jjj-th Gaussian distribution for the iii-th data point. This is the posterior probability of the jjj-th Gaussian distribution given the iii-th data point. The value of wj(i)w_j^{(i)}wj(i)​ is calculated as follows:

q(zi)=wj(i)=ϕjN(xi;μj,Σj)∑k=1KϕkN(xi;μk,Σk)(28) q(z_i) = w_j^{(i)} = \frac{\phi_j \mathcal{N}(x_i; \mu_j, \Sigma_j)}{\sum_{k=1}^K \phi_k \mathcal{N}(x_i; \mu_k, \Sigma_k)} \tag{28} q(zi​)=wj(i)​=∑k=1K​ϕk​N(xi​;μk​,Σk​)ϕj​N(xi​;μj​,Σj​)​(28)

Because we do not know ϕj\phi_jϕj​, this parameter is prior to the EM algorithm. We could initialize it with some random values. In the following code, we initialize ϕj\phi_jϕj​ with 1/K1/K1/K.

class GMM:
    """
    Gaussian Mixture Model with EM algorithm
    
    It is a semi-supervised learning algorithm, which means user need to provide 
    the number of clusters.
    """
    
    def __init__(self, X, k=2):
        # set x as array
        X = np.array(X)
        self.n, self.m = X.shape  # n: sample size, m: feature size
        self.data = X.copy()
        self.k = k  # number of clusters
        
        # initialize parameters for EM algorithm
        
        # initialize the mean vector as random vector for each cluster
        self.mean = np.random.rand(self.k, self.m)
        # initialize the covariance matrix as identity matrix for each cluster
        self.sigma = np.array([np.eye(self.m)] * self.k)
        # initialize the prior probability as equal for each cluster
        self.phi = np.ones(self.k) / self.k
        # initialize the posterior probability as zero
        self.w = np.zeros((self.n, self.k))
        
    def _gaussian(self, x, mean, sigma):
        
        pdf = sp.stats.multivariate_normal.pdf(x, mean=mean, cov=sigma)
        
        return pdf
        
    
    def _e_step(self):
        # calculate the posterior probability based on equation (28)
        for i in range(self.n):
            density = 0 # initialize the density
            for j in range(self.k):
                temp = self.phi[j] * self._gaussian(self.data[i],
                                                        self.mean[j],
                                                        self.sigma[j])
                # update the density (marginal probability)
                density += temp
                # update the posterior probability (joint probability)
                self.w[i, j] = temp
            # normalize the posterior probability
            self.w[i] /= density
            # assert the sum of posterior probability is 1
            assert np.isclose(np.sum(self.w[i]), 1)
            
    def _m_step(self):
        # update the parameters
        for j in range(self.k):
            # get the sum of posterior probability for each cluster
            sum_w = np.sum(self.w[:, j])
            # update the prior probability based on equation (27)
            self.phi[j] = sum_w / self.n
            # update the mean vector based on equation (23)
            self.mean[j] = np.sum(self.w[:, j].reshape(-1, 1) * self.data,
                                                    axis=0) / sum_w
            # update the covariance matrix based on equation (24)
            self.sigma[j] = np.dot(
                    (self.w[:, j].reshape(-1, 1) * (self.data - self.mean[j])).T,
                                (self.data - self.mean[j])) / sum_w
            
    def _fit(self):
        self._e_step()
        self._m_step()
        
    def loglikelihood(self):
        # calculate the loglikelihood based on equation (21)
        ll = 0
        for i in range(self.n):
            temp = 0
            for j in range(self.k):
                temp += self.phi[j] * self._gaussian(self.data[i],
                                                        self.mean[j],
                                                        self.sigma[j])
            ll += np.log(temp)
            
        return ll
    
    def fit(self, max_iter=100, tol=1e-6):
        # initialize the loglikelihood
        ll = [self.loglikelihood()]
        # initialize the number of iteration
        i = 0
        # initialize the difference between two loglikelihood
        diff = 1
        # iterate until the difference is less than tolerance or reach the max iteration
        while diff > tol and i < max_iter:
            # update the parameters
            self._fit()
            # calculate the loglikelihood
            ll.append(self.loglikelihood())
            # calculate the difference
            diff = np.abs(ll[-1] - ll[-2])
            # update the number of iteration
            i += 1
            # print the loglikelihood every 2 iterations
            if i % 2 == 0:
                print("Iteration: {}, loglikelihood: {}".format(i, ll[-1]))

Summary and reflection

I hope after reading this article, you can understand the EM algorithm better. The EM algorithm is a Bayesian algorithm. It is a Bayesian algorithm because it uses the posterior probability to update the parameters.

Let’s take a look at our model again:

p(x,z;Θ), p(x, z; \Theta), p(x,z;Θ),

where Θ=(μ,Σ,ϕ)\Theta = (\mu, \Sigma, \phi)Θ=(μ,Σ,ϕ) is the parameter set, zzz is the random variable that represents the cluster, and xxx is the random variable that represents the data. If you print out the posterior probability www in our GMM example, you will find that the posterior probability is close to 1 or 0, which is just a one-hot vector that maps the data to the cluster. At the same time, ϕ\phiϕ is the prior probability of each cluster.

At the beginning, we initialize ϕ\phiϕ as a uniform distribution, which means that we assume that each cluster has the same probability. Then, we use the posterior probability to update ϕ\phiϕ.

Although we use a simple example to explain the EM algorithm, the Bayesian idea behind this algorithm is very important. If you understand this example well, then you will have a better understanding of the latent variable model.
